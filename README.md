# AONT

## 概要
この PoC は、AES と All-Or-Nothing Transform (AONT)による秘密分散を組み合わせ、データを安全に暗号化・復号する簡易的な機能を提供する。
対象とした単一のファイルを暗号化された任意の個数のファイルに分割し、それらがすべて揃った場合にのみ復号することができる。
ランダム鍵は分割されたファイルすべてから計算可能であるため、AONT は鍵の管理を必要としない。
この PoC ではハッシュとして SHA-256 を使用し、ランダム鍵の鍵長は 32 bytes とする。
本来 AONT において擬似メッセージは暗号化アルゴリズムへの入力として使用されるが、この PoC は AONT そのものの技術的な検証を目的としているため、この PoC では擬似メッセージを対象とした暗号化を扱わない。

| 目的 | コマンド |
| --- | --- |
| 暗号化および秘密分散 | `AONT.exe encrypt -i <source-file> -n <split-number>` |
| 復号 | `AONT.exe decrypt -i <encrypted-file-list> -o <output-file>` |

## 擬似メッセージの構成
1. 平文 (16 byte * N)
2. 元のファイルサイズ (32 bytes)
3. 初期化ベクトル (16 bytes)
4. ハッシュ値(ランダム鍵と「1 から 2 のすべてのブロックのハッシュ」の XOR) (32 byte)

ただし、1 から 2 は、ランダム鍵で暗号化されたそのブロックのインデックスと XOR 演算することにより暗号化されている。
各ブロックのインデックスは、CBC モードの AES により暗号化される。
初期化ベクトルはハッシュ値との XOR 演算により暗号化せず、擬似メッセージに直接含まれている。
その理由は、攻撃者が初期化ベクトルのみを入手しても攻撃に利用することはできず、また仮に初期化ベクトルを暗号化した場合、ランダム鍵で暗号化された初期化ベクトルのブロックのインデックスを算出することが不可能になるためである。
「1 から 2 のすべてのブロックのハッシュ値」それぞれと「4」の XOR 演算の結果は、ランダム鍵と一致する。

## 課題と展望
開発の過程で、データを暗号化したまま計算する秘密計算の機能を拡張することが検討された。
しかし、AES をはじめとする、Sボックスなどの非線形変換を含む暗号化方式では、準同型性が失われ、秘密分散を実行ことは可能である一方、秘密計算を実行することは不可能である。
秘密計算を実行するには暗号化方式 E が

```
E(A) + E(B) = E(A + B)
E(A) * E(B) = E(A * B)
```

を満たさなければならない。
加算や乗算は比較的容易に秘密計算を実行することが可能という話はおそらくここから来ているのだろう。
暗号化されたデータに対して計算を実行するためには、[OpenFHE](https://github.com/openfheorg/openfhe-development) などの完全準同型暗号を使用する必要がある。
大まかに、準同型性は下記に分類される。

* 乗法準同型性
	* 暗号化したまま乗算が可能な性質
* 部分準同型暗号
	* 暗号化したまま加算と乗算が可能だが、乗算の回数に制限がある性質
* 完全準同型性（FHE: Fully Homomorphic Encryption）
	* 暗号化したまま加算と乗算を任意の回数組み合わせることができる性質

論理回路やブール代数に基づいて多項式に変換することで、完全準同型性を満たす暗号化方式で暗号化された暗号文それぞれを対象に計算し、その結果を復号することにより秘密計算を実装することができる。
仮に今後、秘密計算の機能を拡張するのであれば、採用する暗号化方式から AES を排除し、OpenFHE などの完全準同型暗号ライブラリを採用し、計算を加算や乗算の組み合わせに置換して実行するよう、再実装する必要がある。
